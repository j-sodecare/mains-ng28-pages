# mains-ng28-pages

本仓库用于归档和发布多个独立 HTML 页面。所有页面均以静态文件形式存放，不依赖后端服务，适用于直接托管或通过 GitHub Pages 等静态站点服务访问。

## 项目简介

这是一个通用型 HTML 页面归档仓库，用于集中管理、版本控制及发布多个独立页面。仓库本身不针对特定域名或网站，仅为页面提供存储与分发的基础结构。

## 目录说明

```
.
├── index.html          # 根目录入口页面（可选）
├── pages/              # 存放各独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 共用资源文件（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：主要页面存放目录，每个 HTML 文件代表一个独立页面。
- `assets/`：存放页面共用的静态资源，可按需组织子目录。

## 页面归档说明

- 每个页面均为完整的独立 HTML 文件，包含必要的样式与脚本引用。
- 页面之间不共享运行时状态，互不依赖。
- 文件名应具有描述性，避免使用默认名称（如 `page1.html`），建议使用含义清晰的英文或拼音命名。
- 如需添加新页面，直接在 `pages/` 目录下创建 HTML 文件，并确保内部资源路径正确。

## 维护说明

- 本仓库以归档和发布为主要目的，不保证所有页面长期可用或更新。
- 提交新页面时，请保持目录结构清晰，避免冗余文件。
- 如需引用仓库内资源，请使用相对路径，以确保在不同部署环境下正常工作。
- 若页面涉及外部资源（如 CDN 库），请注明来源及版本，便于后续维护。

## 许可

本仓库内容仅供归档与展示，具体页面可能包含各自独立的许可声明。请在使用或分发前确认相关页面的授权情况。
